## Scope

M4 delivers temporal decoupling, the two PEQs, the AT four-phase `nb_transport`
path, the LT↔AT adapters, convenience sockets, and a SIMPLIFY DMI — primarily in a
**new L5 crate `systemrs-tlm-utils`**, with **strictly-additive** extensions to
`systemrs-tlm2` and **exactly one audited kernel primitive** (`Ctx::spawn_thread`,
needed for the AT→LT adapter's per-transaction coroutine). The forward LT path
(rv32i 55/5050, platform, memory) is **frozen and bit-identical**.

## Exit criteria

- **E1** — an LT initiator runs ahead and syncs on grid boundaries; sync count matches a hand calc → M4-04.
- **E2** — a full BEGIN_REQ→END_RESP exchange exercises all three `TlmSync` paths → M4-08 + M4-13.
- **E3** — LT-initiator ↔ AT-target → M4-09.
- **E4** — AT-initiator ↔ LT-target → M4-10.
- **E5** (project-level) — two same-time zero-delay PEQ notifications fire one delta apart, FIFO → M4-05.

The synthesized plan's M4-10 said 'clone the Txn (Rc clone)' into the body — impossible as written.
  - **Mitigation:** CLOSED in M4-10: adapter state (TxnId->Txn, TxnId->EventId) is registered as a Sim service Rc<RefCell<AtToLtState>>; the spawned body captures ONLY the Copy+Send TxnId and re-fetches the Txn via Ctx::current().service::<RefCell<AtToLtState>>() keyed by that TxnId — exactly the pattern add_thread proves (sim.rs:198-201) and the runtime tests document (stackful.rs:54-56). M4-02's test set proves a spawned body can reach per-spawn service state by Copy id; M4-10's two-concurrent-transactions test proves it end-to-end with no state bleed.
